# 📌 Blog / Social Network (React + Firebase + Supabase)

Современное веб-приложение в формате блога / мини социальной сети, разработанное как портфолио-проект для демонстрации навыков frontend-разработки.

Проект приближен к продакшн-уровню по UI/UX, архитектуре и работе с данными.

---

## 🎯 Цель проекта

Основная цель проекта — показать навыки разработки на React:

- работа с реальными backend-сервисами
- проектирование UI/UX интерфейсов
- управление состоянием приложения
- работа с изображениями и оптимизация загрузки
- построение архитектуры, близкой к реальным продуктам

---

## ⚙️ Функционал

### 🔐 Авторизация

- Регистрация и вход пользователей через Firebase Authentication
- Сохранение данных пользователя в Supabase

---

### 📝 Посты

- Создание постов с текстом
- Добавление изображений (от 1 до 10 изображений на пост)
- Оптимизация изображений перед загрузкой (сжатие + конвертация)
- Глобальный лоадер при создании поста

---

### 👤 Профиль пользователя

- Редактирование профиля (имя, описание)
- Система аватарки:
  - добавление / удаление / замена
  - предпросмотр перед сохранением
  - кроп изображения (обрезка под круглый формат)
- Логика отображения кнопок зависит от наличия аватарки

---

### ❤️ Взаимодействие с постами

- Лайки постов
- Сохранение постов
- Отдельные разделы:
  - понравившиеся посты
  - сохранённые посты

---

### 🎨 UI / UX

- Skeleton-загрузки при подгрузке данных
- Алерты (ошибки, успех, уведомления)
- Глобальные лоадеры (например, при входе или создании поста)
- Анимации (Framer Motion)
- Переключение видов ленты:
  - сетка (grid)
  - вертикальный список

---

### 📱 Адаптивность

- Полностью адаптивный интерфейс
- Мобильная версия с нижней навигацией
- Оптимизация под разные экраны

---

## 🧠 Архитектура и особенности

- Используется Redux Toolkit для управления состоянием
- Реализована нормализация и частичное кэширование постов
- Работа с API через Fetch (вместо SDK Supabase в части логики)
- Обработка и оптимизация изображений перед загрузкой
- Разделение логики на Redux слайсы
- Частичная подготовка к масштабированию (чат, подписки, уведомления)

---

## 🚧 Известные ограничения

Проект находится в стадии развития, поэтому:

- нет пагинации (все посты загружаются сразу)
- архитектура местами требует рефакторинга
- используется гибридная связка Firebase + Supabase (временно)
- кэширование реализовано вручную

---

## 🧭 Планируемые улучшения

- Реализация чата в реальном времени (Supabase Realtime → возможно Socket.io)

## 🚀 Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/kapssulka/Blog.git
cd Blog
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка переменных окружения

В проекте есть файл:

```bash
.env.example
```

Скопируй его:

```bash
cp .env.example .env.local
```

И заполни необходимые переменные:

- Firebase ключи
- Supabase ключи

⚠️ Без них проект не запустится, так как они используются для авторизации, базы данных и хранения изображений.

### 4. Запуск проекта

```bash
npm run dev
```

После этого проект будет доступен на локальном сервере
